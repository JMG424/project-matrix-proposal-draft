5Abel — Start with packaging procurement, expand to support (TBD)

Workflow #1: Packaging procurement auto-trigger

Tracking sheet has triggers. When inventory crosses threshold, the agent reads the sheet, identifies the SKU, calculates reorder quantity from velocity, drafts the message to the China team, surfaces to Abel for approval. After 2–3 manual approvals, the system suggests automation. Spans Path, white-label, Everest, Vysta. Abel walked us through this exact flow — we're building what he already designed.

Workflow #2: Customer service triage

Sentiment + topic classification on Zendesk tickets, customer health-score correlation, routing to Annette / Tylor / escalation. Sets up the multi-tenant scale on Quest SaaS without exploding support.

Workflow #3: Ship Day routing

If scope allows: route optimization for local delivery dispatch. Build pattern per workflow: spec → staging → MVP → production. He picks Week 1.

6Matrix dashboard + operating rules

Executive view

Priorities, decisions, risks, open loops, Morpheus alerts, Architect intel, and agent output on one screen.

Sales + finance view

Tylor pipeline, lifecycle segments, revenue brief, customer movement, collections, and outreach status.

Operations view

Tank workflows, procurement alerts, support signals, inventory risk, and implementation status. Read-first by design.

Working principles: Read-only first. Humans approve sensitive actions. Lanes stay separate unless deliberately bridged.
