Phase 2: evolve what we learned into a safer operating layer.

In Phase 1 we built an agent layer to support Oscar, Abel, and Tylor. In practice, that initial design was not ideally suited to how Abel and Oscar actually work day-to-day. Phase 2 evolves that agent layer so it becomes safer, more useful, and has deeper reach into the data that matters most to the organization.

1What this sprint focuses on

Security before scale

Improve access, credentials, monitoring, and change-control while staging NEMOClaw-style sandbox environments so Path agents can be tested safely before deeper integrations.

Company source of truth

Begin organizing entities, people, systems, rules, decisions, and context into a cleaner shared operating map.

Workflow integration

Identify the workflows where agents can safely draft, summarize, monitor, and support the team.

4Abel — Start with the workflows that hurt

For Abel, we start narrow: choose one or two painful workflows, build safely, and only expand once the pattern proves useful.

Focused starting point

  • Pick 1–2 workflows: likely inventory/procurement, support intelligence, or meeting-to-task execution.
  • Use drafts and alerts first: the system recommends; people approve.
  • Build around real pain: no broad automation menu until the first workflow is working.

Safe build pattern

  • Spec: data sources, permissions, failure modes, approval rules.
  • Sandbox: test in staging before production data or credentials.
  • MVP: limited production use with monitoring and manual approval.
  • Extend: document what works before expanding to the next workflow.

5Matrix dashboard + agent interface

The dashboard and agent interface support Oscar in using AI to see the company’s operations clearly, on demand, without needing to chase every update through people and tools.

Executive view

Priorities, decisions, open loops, agent output, risks, and next actions across the operating system.

Sales + finance view

Tylor pipeline, lifecycle segments, revenue brief, customer movement, and outreach status.

Operations view

Abel workflows, procurement alerts, support signals, inventory risk, and implementation status.

Deliberate constraint: dashboard v1 is read-first. It gives Oscar visibility and review context. Production changes still go through staging, approval, and Alacrity One change-control.

6Working principles

Read-only first

The agents should observe, summarize, draft, and recommend before they are allowed to change anything.

People approve sensitive actions

Anything outbound, financial, customer-facing, regulatory, or production-related stays under human approval.

Keep lanes clear

Sales, finance, supplier, compliance, and operations context stay separate unless there is a clear reason to connect them.
